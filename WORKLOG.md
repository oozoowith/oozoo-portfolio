## 6. 이미지 추가 방법 (드롭인)

이미지가 없으면 회색 placeholder가 자동 표시됩니다. 아래 경로에 파일만 넣으면 자동으로 나타납니다(코드 수정 불필요).

```
images/
├── works/
│   ├── moints/  gongil/  wal/  ccbook/  ccin/  sqnc/  txtclub/  frip/
│   │     └── cover.webp (필수, 가로 3:2) · 01.webp · 02.webp (16:9)
```

## 6.5 프로젝트 상세 페이지 구조 (2026-06-20 개편, 8개 전체 적용)

클릭 시 전체화면 상세. 3개 섹션:
1. **소개** — 제목 / 한줄설명(tagline) / 커버 / 프로젝트 소개(intro) / 메타 한 줄(소속·기간·카테고리·역할(기여도는 줄바꿈)) + 수행 업무(전체폭)
2. **내용** — 목표 / 타겟 / 전략(✶ 핵심기준 `strategyCore` + 상세 `strategy`는 PDF 원문, lv1·lv2 들여쓰기) / 성과(`results` 숫자 카드, 가운데 정렬·테두리 구분; 부가 설명은 `resultNote`)
3. **사진** — '프로젝트 살펴보기'. 그룹별 소제목+설명+사진. 사진 개수에 따라 자동 그리드(1=풀폭, 2=2단, 3=3단…).

### 사진 자동 추가/교체 (중요)
- 각 그룹은 **접두어**(`prefix`)를 가짐. 보통 `a`=현장 사진(가로), `b`=디지털·콘텐츠(세로).
- JS가 `images/works/{폴더}/{접두어}1.webp, {접두어}2.webp …` 를 1번부터 순서대로 자동 탐지. **빠진 번호에서 멈춤**.
- **사진 추가**: 다음 번호 파일을 폴더에 넣고 새로고침 → 자동 표시 + 레이아웃 자동 조정. (예: `a5.webp` 추가)
- **사진 교체**: 같은 이름으로 덮어쓰기.
- 사진 0장인 그룹은 자동 숨김. 그룹 자체를 추가/제목·설명 변경은 index.html의 해당 프로젝트 `photos` 배열 수정.
- 커버는 그대로 `cover.webp` (3:2).
